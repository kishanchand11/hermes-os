# Part VI – skills.sh

# 27. Capability Gap Analysis

## 27.1 Purpose

Capability gaps identify missing abilities required for engineering objectives.

## 27.2 Evaluation Rule

Hermes must first evaluate existing agency capabilities before seeking additional skills.

## 27.3 Gap Process

Need identified → Existing capabilities reviewed → Gap confirmed → Solution evaluated

## 27.4 Restrictions

Additional tools must never be introduced without verified need.
